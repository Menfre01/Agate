# AI Gateway - Claude 开发规范

## 提交规范

### Commit 前必须完成单元测试

在执行 `/commit` 或创建 git commit 之前，**必须**为新编写的代码添加基本的单元测试。

**验收标准**：
- 新增的核心函数/方法必须有对应的单元测试
- 测试覆盖正常路径和基本错误情况
- `npm test` 必须通过
- 复杂逻辑的覆盖率应达到合理水平

**例外情况**：
- 类型定义文件（`*.d.ts`）
- 配置文件（JSON、JSONC）
- `.gitkeep` 等占位文件

---

## 开发约定

### 代码风格
- 使用 TypeScript 严格模式
- 所有公共函数必须添加 JSDoc 注释
- 导出的类型和接口必须有说明注释

### Git 工作流
1. 完成功能开发
2. 编写/更新单元测试
3. 运行 `npm test` 确保通过
4. 运行 `npm run typecheck` 确保类型正确
5. 使用 `/commit` 提交代码
